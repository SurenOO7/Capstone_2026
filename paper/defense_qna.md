# Պաշտպանության հնարավոր հարցեր և պատասխանների հենք

## Ինչու ընտրվեց անձնական նպատակների հետևման թեման

Թեման թույլ է տալիս ցույց տալ առաջատար վեբ ծրագրավորման ամբողջական կիրառություն՝ տվյալների բազա, նույնականացում, պաշտպանված էջեր, ինտերակտիվ միջերես, responsive դիզայն և վերլուծական ցուցադրումներ։ Բացի այդ, այն ունի գործնական խնդիր՝ նպատակների կառավարման մասնատվածությունը։

## Ինչով է լուծումը տարբերվում Todoist-ից կամ Habitica-ից

Todoist-ը հիմնականում task management գործիք է, Habitica-ն՝ gamification-ի վրա հիմնված լուծում, իսկ մշակված կայքը կենտրոնացած է չափելի նպատակների, ենթանպատակների, առաջընթացի պատմության և analytics-ի ինտեգրման վրա։ Նպատակը ոչ թե առևտրային գործիքներին փոխարինելն է, այլ թեմատիկ կայքի ամբողջական և ընդլայնելի մոդել ցույց տալը։

## Ինչու ընտրվեցին Next.js, Prisma և PostgreSQL

Next.js-ը թույլ է տալիս մեկ նախագծում համադրել frontend և server-side գործողությունները, Prisma-ն ապահովում է type-safe տվյալների մուտք և schema-first մոտեցում, իսկ PostgreSQL-ը հարմար է հարաբերական տվյալների ամբողջականության, կապերի և ինդեքսների համար։

## Ինչպես է պահպանվում օգտատիրոջ տվյալների պաշտպանությունը

Protected route-երը պահանջում են session, server action-ները ստուգում են ընթացիկ օգտատիրոջը, իսկ goal փոփոխելուց առաջ կատարվում է ownership ստուգում։ Credentials մուտքի դեպքում գաղտնաբառը պահվում է hash-ավորված տեսքով։

## Որն է աշխատանքի հիմնական սահմանափակումը

Ամենակարևոր սահմանափակումը իրական օգտատերերի վրա լայն usability testing-ի բացակայությունն է։ Այդ պատճառով առաջարկվել է հաջորդ փուլում փորձարկել համակարգը ուսանողների և երիտասարդ մասնագետների փոքր խմբով։

## Ինչ է կարելի զարգացնել հաջորդ փուլում

Անհատականացված շաբաթական վերանայում, կարգավորվող push notification-ներ, calendar integration, Progressive Web App տարբերակ, թիմային նպատակներ և accessibility audit՝ WCAG 2.2 չափանիշներով։
